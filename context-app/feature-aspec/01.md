# Feature: Authentication & Role-Based Access Control

## Overview

Build a complete authentication system with signup and sign-in for Editor and Viewer roles.
The Admin role is created directly in the system (not through signup).
UI design follows Irembo-style modern, clean, professional interface.

---

## Requirements

### 1. User Roles

| Role   | Signup Allowed | Created By     |
|--------|---------------|----------------|
| Admin  | No            | System-seeded  |
| Editor | Yes           | Self-signup    |
| Viewer | Yes           | Self-signup    |

- Admin account is pre-seeded into the database (not created via signup).
- Editor and Viewer users sign up through the registration page.
- Role selection is part of the signup form.

---

### 2. Authentication Pages

#### Sign In Page (`/signin`)
- Clean card-centered layout (Irembo-inspired)
- Email and password fields
- Role-aware redirect after login (Admin → dashboard, Editor/Viewer → projects)
- "Don't have an account? Sign up" link
- Form validation with error messages
- Loading state during submission

#### Sign Up Page (`/signup`)
- Clean card-centered layout (Irembo-inspired)
- Full name, email, password, confirm password, role selection (Editor/Viewer)
- "Already have an account? Sign in" link
- Form validation with error messages
- Loading state during submission
- Success redirect to sign-in page

---

### 3. API Endpoints

#### `POST /api/auth/signup`
- Accepts: name, email, password, role (EDITOR | VIEWER)
- Validates input
- Checks for existing user by email
- Hashes password with bcrypt
- Creates user in database
- Returns success message (do not auto-login after signup)

#### `POST /api/auth/signin`
- Accepts: email, password
- Validates input
- Finds user by email
- Compares password hash
- Generates JWT token with user id, email, role
- Returns token and user info (id, name, email, role)

#### `GET /api/auth/me`
- Protected route (requires valid JWT)
- Returns current authenticated user info
- Used for session validation on page load

---

### 4. Frontend Implementation

#### Auth Context / Provider
- React context to manage auth state globally
- Store JWT in localStorage
- Provide user object, login/logout functions, loading state
- Auto-restore session on app load via `/api/auth/me`

#### Protected Route Logic
- Redirect unauthenticated users to `/signin`
- Redirect authenticated users away from `/signin` and `/signup` to home
- Role-based route access (future)

#### UI Design (Irembo-inspired)
- Clean white card on subtle gray background
- Primary color: `#1E3A8A` (deep blue)
- Rounded corners on cards and inputs
- Professional typography (font-weight 500-600 for labels)
- Proper spacing and padding
- Focus rings on inputs
- Error states with red border and error text
- Loading spinner on buttons during submission
- Smooth transitions and hover effects

### 5. Database Schema (Prisma)

```prisma
model User {
  id        String   @id @default(cuid())
  name      String
  email     String   @unique
  password  String
  role      Role     @default(VIEWER)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

enum Role {
  ADMIN
  EDITOR
  VIEWER
}
```
